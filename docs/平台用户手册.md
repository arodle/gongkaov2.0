# 公考学习平台用户手册

更新日期：2026-05-28

## 1. 平台定位

本平台是一个“学练一体”的公考备考系统，核心思路是：

- 用 MindCanvas 学习知识结构。
- 在知识节点上进入针对性练习。
- 在智能练习页按不同模式刷题。
- 系统根据做题正确率、错题情况和复习间隔，更新每个知识点的掌握状态。
- 数据报告用于复盘能力变化、薄弱点和做题行为。
- 题库管理和导图编辑偏运营端，用于维护知识点、题目和真题套卷。

平台当前适合单用户或小规模运营使用，后续增加完整用户体系、权限分层和多端 App/小程序时，需要继续强化数据隔离、权限和移动端交互。

## 2. 账号与登录

### 登录

打开平台后，未登录用户会进入登录页。输入邮箱和密码即可进入平台。

### 注册

没有账号时，可以切换到创建账号。密码至少 8 位。注册后会自动登录。

### 数据归属

当前平台已经有用户账号边界。知识导图、题库、练习记录、做题行为事件和数据报告都会尽量按当前登录用户隔离。

注意：如果同一浏览器切换不同账号，平台会清理本地 IndexedDB 中上一位用户的数据缓存，避免本地数据串号。

## 3. 主导航

平台主导航包含 6 个入口：

- MindCanvas：学习知识导图和进入 Mind 编辑。
- 智能练习：选择刷题模式并开始练习。
- 错题本：查看错题、编辑错题笔记。
- 题库管理：新增、导入、编辑、删除题目，管理真题套卷。
- 数据报告：查看能力雷达、PS 趋势、全景图和做题记录回放。
- 个人中心：数据导出、恢复、同步和系统配置。

桌面端使用左侧栏导航；手机端使用底部导航。

## 4. MindCanvas 知识学习

MindCanvas 是用户学习知识结构的主入口。

### 浏览导图

可以在画布中拖拽移动视图，使用滚轮或缩放按钮放大缩小。

缩小时，导图会保留根节点和一级主干；放大后更多子级节点会逐步出现。这样可以避免大型导图在小屏上挤成一团。

### 搜索知识点

左上角搜索框可搜索知识节点。点击搜索结果后，画布会定位到对应节点。

### 按真题套卷高亮知识点

如果题库中的题目已经归入真题套卷，例如“2026年国考副省级”，可以在 MindCanvas 的套卷下拉中选择该套卷。平台会把这套题涉及的知识点集中高亮，其它知识点保持默认状态。

适用场景：

- 做完一套真题后，查看它覆盖了哪些知识点。
- 判断某一年考试的知识点分布。
- 针对某套卷暴露出的薄弱点进行复习。

### 节点颜色含义

知识点颜色来自练习结果和复习状态：

- 薄弱：长期未练、做错较多或掌握度低，需要优先复习。
- 需加强：有一定基础，但稳定性不足。
- 学习中：正在积累，表现中等。
- 熟练：近期练习表现较好，暂时不是优先复习对象。

### 节点练习

点击节点后，如果该节点关联了题目，可以进入节点练习。适合针对某一个知识点进行强化。

## 5. Mind 编辑

Mind 编辑是运营端和内容维护入口，也可以用于高级用户整理自己的知识图谱。

### 大纲

左侧大纲以树状文本展示所有节点。

可用操作：

- 点击大纲节点：定位并选中画布节点。
- 双击标题：直接编辑节点名称。
- 点击箭头：折叠或展开子级。
- 拖拽节点：调整父子层级。
- 删除节点：删除当前节点及子级。

### 文本编辑器

选中节点后，可以编辑：

- 标题
- 节点类型
- 摘要内容
- Markdown 正文
- 节点宽度和高度

正文编辑支持标题、加粗、斜体、列表、引用、代码和预览。

### 画布编辑

右侧画布用于观察结构变化。节点右下角可拖动调整矩形大小。

保存前请确认层级结构、节点标题和内容已经正确。点击保存后会写入 Neon 数据库。

### 快捷键

- `/`：聚焦搜索。
- `+ / -`：放大或缩小画布。
- `0`：适配视图。
- `Enter / Space`：折叠或展开当前节点。
- `E`：展开全部。
- `C`：收起到根层级。
- `Delete / Backspace`：删除当前节点。
- `Esc`：取消选择或关闭浮层。

## 6. 智能练习

智能练习用于用户刷题。

### 练习题数

可以选择：

- 全部
- 5 题
- 10 题
- 15 题

后续可以扩展为 10/30/50/100 或按套卷固定题量。

### 答题模式

逐题作答：选择答案后立即反馈对错。

整卷提交：先完成全部题目，再统一提交，适合模拟考试。

### 练习模式

顺序练习：按知识图谱层级依次练习。

随机练习：从题库中随机抽题。

靶向练习：优先练习薄弱知识点。

套卷练习：按真题套卷或完整试卷进行练习。

### 做题行为采集

做题过程中，平台会记录结构化行为事件，不保存截图、不保存画板图片。

已支持事件：

- highlight：高亮题干文字。
- circle：圈选关键词。
- strike：划掉选项。
- answer_select：选择答案。
- answer_change：修改答案。
- note：添加简短备注。

每条事件包含：

```json
{
  "questionId": "",
  "userId": "",
  "eventType": "",
  "target": "",
  "startTime": "",
  "endTime": "",
  "metadata": {}
}
```

这些数据用于后续分析做题习惯，例如是否反复改答案、是否能抓关键词、是否经常排除错误选项。

## 7. 错题本

错题本用于复盘错误题目。

主要用途：

- 查看错题列表。
- 查看题目、选项、正确答案和解析。
- 编辑错题笔记。
- 将错题绑定到知识层级。
- 按错因统计发起专项训练。
- 导出错题数据。

建议用户每次练习后进入错题本，至少完成三个动作：

- 判断错因。
- 写一句错题笔记。
- 点击高频错因做专项回炉。
- 专项训练做对后会自动推进二刷间隔，稳定做对后标记为已掌握。
- 回到 MindCanvas 复习对应知识点。

## 8. 题库管理

题库管理偏运营端，用于维护平台题目。

### 添加题目

可填写：

- 题目类型：真题或模拟题。
- 关联知识点。
- 题干。
- 选项。
- 正确答案。
- 解析。
- 真题套卷。
- 题目出处。
- 解析图片。

### 编辑题目

点击题目右侧编辑按钮，可以修改题目内容。修改后保存即可同步到本地和远端。

### 删除题目

点击删除按钮可删除题目。删除前应确认该题不会影响套卷完整性和历史练习记录。

### 批量归入真题套卷

可以先筛选题目，再选择当前页或全部筛选结果，将它们批量归入某个套卷，例如“2026年国考副省级”。

套卷字段会影响 MindCanvas 中的套卷高亮功能。

### 分页

题库支持 10/50/100 题分页，适合不同规模的题库管理。

### 导入题库

支持导入 JSON 题库文件。导入后应检查：

- 题干是否正常。
- 选项是否完整。
- 正确答案是否匹配。
- 解析是否乱码。
- 知识点绑定是否正确。
- 真题套卷字段是否完整。

### 导出题库

可导出当前筛选结果，用于备份或迁移。

## 9. 数据报告

数据报告用于学习复盘。

### 能力雷达

展示言语理解、数量关系、判断推理、资料分析、常识判断等维度的能力分布。

### PS 趋势

展示 PS 分数随时间变化。可以观察某个知识点或整体能力是否提升。

### 全景图

用图形方式展示知识点掌握分布，帮助快速发现薄弱区域。

### 做题记录

做题记录按题展示历史答题记录。

每条记录可查看：

- 题目内容。
- 选项。
- 是否正确。
- 用时。
- 行为时间轴。
- 备注事件。

### 做题过程回放

时间轴按事件顺序展示做题行为，例如：

- 第 3 秒：高亮“然而”。
- 第 8 秒：划掉 B。
- 第 12 秒：选择 A。
- 第 18 秒：修改为 C。

点击时间轴事件后，页面会定位到题干、选项或备注区域，适合复盘解题过程。

## 10. 个人中心

个人中心用于数据管理。

### 导出数据

可以导出学习数据备份。备份通常包含：

- 知识点数据。
- 练习记录。
- PS 历史。
- 相关配置。

### 从备份恢复

可以导入备份文件恢复数据。恢复会覆盖当前数据，操作前应先导出现有数据。

### 同步

平台支持本地 IndexedDB 和 Neon 数据同步。网络异常时可以先使用本地数据，恢复网络后再同步。

## 11. 手机端使用建议

手机端适合：

- 浏览 MindCanvas。
- 节点复习。
- 做题练习。
- 查看错题和报告。

手机端不太适合：

- 大规模编辑导图。
- 批量维护题库。
- 复杂的数据迁移。

后续如果做 App 或小程序，建议将学习、练习、错题复盘作为用户端核心，把题库管理、导图编辑保留在运营端或桌面端。

## 12. 常见问题

### 为什么某些题显示“未绑定知识点”？

通常是题目没有关联知识点，或者导入时知识点 ID 与当前导图节点不匹配。需要在题库管理中重新绑定。

### 为什么 MindCanvas 选套卷后没有高亮？

可能原因：

- 题目没有填写真题套卷字段。
- 套卷名称不一致，例如“2026国考副省级”和“2026年国考副省级”被视为不同套卷。
- 题目没有绑定知识点。

### 为什么做题记录没有行为回放？

只有在做题时使用高亮、圈选、划掉、修改答案或备注等操作，才会生成行为事件。

### 为什么刷新后数据变化？

平台同时使用本地 IndexedDB 和 Neon。如果同步过程中远端数据覆盖本地，刷新后可能看到远端状态。遇到异常时先不要继续批量编辑，建议导出备份后再排查。

# 内部问题清单：漏洞、bug 与产品风险

以下内容面向开发和运营，不建议直接暴露给普通用户。

## P0：需要优先处理

### 1. 账号注册缺少限制与防刷机制

当前处理状态：已基本完成。注册接口已增加频率限制；生产环境非首个用户默认要求邀请码，除非显式开放公开注册。

现状：注册接口允许用户直接创建账号，未看到验证码、邀请制、频率限制或 IP 限制。

风险：

- 被恶意批量注册。
- 数据库被刷入大量无效账号。
- 后续开放公网后风险明显增加。

建议：

- production 环境默认关闭公开注册，改为邀请码或管理员创建。
- 登录和注册增加 rate limit。
- 失败次数过多时临时锁定。

### 2. 整卷/逐题练习提交存在状态边界风险

当前处理状态：已基本完成。提交时会检查未答题，并从 `userAnswers` 重新计算整卷结果。

现状：逐题模式最后一题的提交按钮在部分情况下可能允许未答题提交；答题结果依赖 React state 异步更新，极端情况下可能漏记最后一次答案。

风险：

- 数据报告正确率不准。
- PS 更新不完整。
- 用户以为已完成，实际最后一题未计入。

建议：

- 最后一题提交前强制检查当前题是否已答。
- 提交时从 `userAnswers` 重新计算结果，而不是依赖 `answers` state。
- 增加练习提交流程测试。

### 3. 层级重建接口具有破坏性

当前处理状态：已基本完成。接口要求 admin、必须指定 `userId`，默认 dry-run；执行写操作需要显式确认，更新和删除均带 `user_id`。

现状：`/api/rebuild-hierarchy` 会按固定层级保留节点，并删除不在固定名单内的节点。虽然要求 admin，但逻辑本身非常危险。

风险：

- 误调用会删除大量真实导图节点。
- 如果未传 `userId`，可能影响同一 `mapId` 下所有用户数据。
- 最后的 `UPDATE map_node SET parent_id = NULL WHERE name = '行测' AND mind_map_id = mapId` 未带 `user_id` 过滤。

建议：

- production 禁用该接口，或只允许本地开发使用。
- 改为 dry-run 预览，确认后再执行。
- 所有更新和删除都必须带 user_id。
- 删除前写入审计日志和快照。

### 4. 备份恢复是覆盖式操作

当前处理状态：已部分完成。恢复前会自动创建本地快照，并对备份结构、版本和 userId 混杂进行基础校验；差异预览仍未完成。

现状：个人中心恢复备份会完全替换当前数据。

风险：

- 用户误操作后丢失最新数据。
- 备份文件跨账号导入可能造成数据混杂。

建议：

- 恢复前自动生成当前快照。
- 导入文件中必须包含 userId、版本号、数据校验。
- 支持“预览差异”再确认恢复。

## P1：重要但可排期

### 5. 套卷名称缺少规范化

当前处理状态：已继续推进。题库保存、批量归入、创建套卷会做基础名称规范化；服务端已新增 `exam_papers` 标准表，同步接口会返回/写入标准套卷目录，题目同步时也会自动维护套卷记录。题目仍保留 `exam_paper` 文本字段用于兼容，完全改为 `examPaperId` 引用仍未完成。

现状：套卷名称依赖文本输入。

风险：

- “2026国考副省级”“2026年国考副省级”“2026 国考 副省级”会变成不同套卷。
- MindCanvas 套卷高亮可能不完整。

建议：

- 已建立 `exam_papers` 独立表，并由同步接口维护。
- 后续将题目引用改为 `examPaperId`，逐步减少自由文本依赖。
- 输入时继续使用标准下拉，并增加更严格的校验。

### 6. 知识点绑定仍可能出现 ID 不一致

当前处理状态：已继续推进。已新增统一的知识点绑定一致性检查器，覆盖 `mn_` 前缀兼容、完整路径匹配、路径末级名称回退和同名节点风险提示；题库管理已增加“绑定异常”统计、筛选和异常原因展示。巡检结果会保存为完整报告，每日自动刷新一次，并支持导出巡检报告。题库管理已提供知识点别名库，可维护“常见叫法 -> 标准知识点路径”的映射，并参与草稿题自动匹配。统一主键策略仍未完全落地。

现状：新旧表、导入题库、迁移数据之间存在 nodeId/legacyId/linkedAngleId 的兼容逻辑。

风险：

- 导入后题目显示未分类。
- 套卷高亮无法命中节点。
- 练习后 PS 更新到错误节点。

建议：

- 继续统一知识点主键策略。
- 已提供绑定一致性检查器和每日自动巡检报告，后续可扩展为后台巡检脚本。
- 题库管理已显示“绑定异常”筛选和异常原因。
- 题库管理已提供知识点别名库，后续可迁移到服务端表并做多用户共享。

### 7. 做题行为事件缺少版本与 schema 校验

当前处理状态：已继续推进。行为事件已增加 `schemaVersion`，并按事件类型规范化 metadata：文字高亮/圈选校验偏移量，划掉选项校验选项标签，答题事件校验答案字段，备注事件校验备注文本；本地记录、同步入库、查询和回放都会走同一套规范化逻辑。后续可继续把 schema 暴露为后台巡检报告。

现状：事件 metadata 是开放对象，便于扩展，但缺少版本字段和校验。

风险：

- 后续回放逻辑升级后旧事件不兼容。
- metadata 字段缺失时前端回放异常。

建议：

- 已增加 `schemaVersion`。
- 已按 eventType 规范化和校验 metadata 基础字段。
- 回放组件读取旧事件时会先规范化再展示。

### 8. 图片存储方式可能过重

当前处理状态：已继续推进。知识点笔记截图已限制 2MB 以内；题库图片上传、题库 JSON 导入、套卷导入和 store 写入层已增加统一图片限制：单张 inline 图片不超过 2MB，每题最多 6 张，过大的 data URL 会被跳过。对象存储、压缩上传和 URL 化仍未完成。

现状：解析图片和预览可能以 base64/data URL 形式进入题库。

风险：

- IndexedDB 和同步 payload 过大。
- Neon 存储压力上升。
- 页面加载变慢。

建议：

- 后续图片改为对象存储，例如 Vercel Blob、S3、R2。
- 数据库只保存 URL 和元信息。
- 已限制 inline 图片体积和数量，后续继续增加上传压缩。

### 9. 删除题目和删除节点缺少影响提示
