# 公考学习平台设计规范

## 目标

平台 UI 优先服务长期学习与运营维护，风格应当稳定、清晰、可扫描。学习端减少干扰，运营端强调密度、批量操作和风险提示。

## 信息架构

- 学习端：MindCanvas、智能练习、错题本、数据报告。
- 运营端：题库管理、Mind 编辑、备份恢复、套卷维护。
- 系统端：账号、同步、快照、最近删除。

移动端默认优先学习端；运营端在移动端保留入口，但必须提示建议桌面端操作。

## 色彩

- 主色：`#a49aff`，用于主操作、选中态和关键导航。
- 辅助蓝：`#b7e3ff`，用于轻量高亮和信息提示。
- 风险红：`#dc2626` / `#fee2e2`，用于薄弱点、删除和高风险提示。
- 高频琥珀：`#d97706` / `#fef3c7`，用于高频考点、套卷覆盖和待处理提醒。
- 中性色：`slate` 系列，用于文本、边框、面板背景。

避免大面积高饱和渐变；运营页优先白底、浅灰底和低饱和提示色。

## 组件规则

- 卡片半径控制在 `8px` 左右，避免卡片套卡片。
- 工具条使用轻量浮动面板，图标按钮优先；文字按钮只用于明确命令。
- 表单字段高度保持 `32-36px`，批量操作区使用浅色警示底。
- 删除、恢复、覆盖式导入必须展示影响范围或风险说明。
- 过滤器优先使用 segmented control 或 select；数量徽标放在筛选项内或统计栏。

## MindCanvas

- 默认显示完整学习结构，薄弱/高频过滤只收束视觉，不改变数据。
- 空白画布默认箭头光标；只有拖动时显示抓取光标。
- 节点颜色遵循掌握度：红色需加强、琥珀学习中、绿色熟练、灰色待学习。
- 工具条第一排放搜索、聚焦过滤、套卷；第二排放布局、连线和颜色。

## 题库管理

- 搜索、类型筛选、绑定异常筛选必须常驻顶部。
- 批量归入套卷必须使用规范化后的套卷名。
- 删除题目前展示历史答题、套卷和知识点绑定影响。
- 移动端提示建议桌面端操作。

## 数据报告

- 报告页必须解释 PS、薄弱点、雷达图和行为回放口径。
- 图表不直接等同考试分数，文案中应避免造成误解。
- 行为回放只展示结构化事件，不展示截图或敏感原始操作画面。

## 安全与恢复

- 覆盖式恢复前必须自动创建快照。
- 最近删除保留最近 20 条，可用于误删恢复。
- 生产环境注册默认要求邀请码，除非显式开放公开注册。
